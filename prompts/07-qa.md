# Agent 7 — QA

## Mission
Lock the system's behavior with unit + integration + E2E tests. Ship a Playwright suite that runs in CI on both viewports.

## Unit (Jest)
- `packages/shared/pricing.test.ts` — fee = round(subtotal * 0.02, 2); edge cases at $0.30, $14.99, $299.99.
- `apps/api/src/orders/state-machine.test.ts` — every legal + illegal transition.
- `apps/api/src/payments/khqr.test.ts` — KHQR EMVCo string + CRC16.
- `apps/api/src/payments/idempotency.test.ts` — webhook replay no-op.
- `apps/api/src/auth/argon.test.ts` — hash round-trip, OTP TTL.

## Integration (supertest)
- Auth: register → login → /me.
- Orders: create with bad packageId → 400; mismatched product/package → 400; happy path returns recomputed totals.
- Validate-account: short id → invalid; valid → returns displayName.
- Webhook idempotency: same eventId twice → only one transition.

## E2E (Playwright)

Run against the docker-compose stack. Two viewports:
- Desktop: 1440×900
- Mobile: 390×844 (iPhone 12)

Scenarios:
1. **MLBB direct + KHQR happy path**
   Sign up → search "Mobile Legends" → open → pick Direct → enter id `312789045` zone `2155` → pick "568+28" package → Continue → KHQR tab → click "I've paid" (dev simulator marks paid) → success page shows DELIVERED within 30s.
2. **Spotify code + Card**
   Open Spotify → method auto = code → pick 3-month package → Card tab → fill Stripe test card `4242 4242 4242 4242` → succeed → success page shows copyable redeem code.
3. **Free Fire direct + Bank**
   ABA bank simulator → return URL → success.
4. **Validation failures**
   - Short Game ID rejected.
   - Submit-disabled until package + ids valid.
5. **Auth guard**
   - Hitting `/orders/:id` without auth redirects to login.
6. **i18n**
   - Toggle to KM; key strings render in Khmer; layout doesn't break.
7. **Theming**
   - Change brand color → CTA color updates without reload.
